# Stream
|Mode|流式传输的内容|触发时机/来源|主要用途|
|--|-|-|-|
|updates|工作流的状态更新|每完成一个代理步骤|监控工作流的进度和状态
|messages|LLM 生成的 (token, metadata)|图中调用 LLM 的节点|实时查看 LLM 文本输出
|custom|任意自定义数据|图节点内通过流编写器触发|传输特定、非标准的中间数据

## updates
Agent Progess
这个模式关注于工作流状态的实时变化。每执行一个代理步骤后，就会发出一个事件
适合开发者

## messages
LLM tokens
这个模式关注于大模型的输出
就像用户在使用豆包一样，可以看到豆包每一步的输出
适合作为发行版为用户使用

## custom
Custom Updates
这个模式提供了最大的灵活性，允许用户定义自己的流数据
当您想在工作流执行过程中，流式传输非状态更新和非 LLM 令牌的特定信息时（例如，中间计算结果、特定的日志信息、调试数据等），就可以使用此模式。
适合高级开发者使用
